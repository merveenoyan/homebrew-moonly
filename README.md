# 🌙 Moonly

A macOS **menu-bar** cycle tracker. Log symptoms in a tap, get phase-aware
recommendations from a model that runs **entirely on your Mac**, and never send
a single byte of health data anywhere.

> **Privacy is the product.** Your logs live in a local JSON file. Suggestions
> are generated by llama.cpp running **in-process** (via
> [LlamaKit](https://github.com/pcuenca/LlamaKit)). The only network request
> Moonly ever makes is the one-time, inbound download of the model from Hugging
> Face.

---

## What it does

1. **Log symptoms** — period + flow, a curated symptom set, mood, and energy,
   all from the menu-bar dropdown.
2. **Tracks your cycle** — learns your average cycle and period length from your
   own history and shows the current phase, cycle day, fertile window, and
   predicted next period.
3. **Predicts and recommends as the month progresses** — forecasts upcoming
   phase transitions ("Luteal begins in 2 days") and tells you what to do *today*
   (deep work, socialize, rest, journal, stay in…). An instant rule-based
   baseline per phase is upgraded in place by **Gemma 4 E4B (QAT)**, which is fed:
   - your **last three period start dates** and today's date,
   - the **predicted phase forecast** for the days ahead, and
   - **what you felt at this same point last cycle** (e.g. irritable + cravings),
     so it can anticipate today rather than just describe it.
4. **Stays on-device** — no accounts, no telemetry, no cloud.
5. **Installs with Homebrew.**

## The model

Moonly serves **[`google/gemma-4-E4B-it-qat-q4_0-gguf`](https://huggingface.co/google/gemma-4-E4B-it-qat-q4_0-gguf)**:

- **Official Google QAT checkpoint** — quantization-aware training recovers the
  accuracy that naive Q4_0 conversion loses, so a ~5 GB Q4_0 file punches well
  above its weight.
- **Apache-2.0** — clean to redistribute and ship in a product.
- **E4B** — the larger of the two Gemma 4 "effective-param" nano variants;
  strong instruction following at a size that runs comfortably on Apple Silicon.

The GGUF is loaded **in-process** by LlamaKit (Swift bindings for llama.cpp).
On first launch it's downloaded from the Hugging Face Hub into
`~/Library/Application Support/Moonly/models` (via the `HF_HUB_CACHE` env var) so
it's app-owned and removed on uninstall. (Runner-up if you ever want marginally
better quality at the same size: `unsloth/gemma-4-E4B-it-qat-GGUF:UD-Q4_K_XL`.)

## Architecture

```
MenuBarExtra (SwiftUI)
        │
        ├── CycleStore ......... logs + cycle math, JSON in Application Support
        ├── RecommendationEngine  rule baseline → personalized via the model
        └── LlamaServer ........ loads the GGUF in-process via LlamaKit,
                                 applies Gemma's chat template, generates locally
```

The engine is **bundled**: LlamaKit links llama.cpp as an embedded
`llama.framework` inside the app, so there's no external `llama.cpp` install to
manage.

## Install (Homebrew)

```bash
brew install --cask ./Casks/moonly.rb     # or from a tap once published
```

This installs the app and, if the `hf` CLI is present, pre-downloads the model
(~5 GB, one time). If the pre-download is skipped or interrupted, the app fetches
the model on first launch and shows progress in the dropdown.

> The model download exceeds GitHub Releases' 2 GB asset limit, so the cask's
> `url` points at a Hugging Face–hosted DMG. The DMG itself stays small — only
> the model is large, and it's fetched separately on first launch.

## Build from source

Requires macOS 14+ with the **Swift 6.1+** toolchain (Xcode Command Line Tools);
LlamaKit uses package traits, which need that toolchain.

```bash
Scripts/build_app.sh      # → dist/Moonly.app
Scripts/make_dmg.sh       # → dist/Moonly.dmg (+ prints version & sha256)
```

Paste the printed `sha256` and bump `version` in `Casks/moonly.rb`.

For real distribution, replace the ad-hoc `codesign` in `build_app.sh` with a
Developer ID identity and notarize the DMG.

## Where your data lives

| What | Path |
|------|------|
| Symptom logs | `~/Library/Application Support/Moonly/data.json` |
| Cached model | `~/Library/Application Support/Moonly/models/` |

`brew uninstall --cask moonly` + `--zap` removes both.

## Disclaimer

Moonly offers general lifestyle and wellness suggestions and is **not medical
advice**. Predictions are heuristic estimates from your logged history, not a
contraceptive method. See a clinician for severe, unusual, or persistent
symptoms.
